Reactjs và cấu trúc dự án

React là một thư viện JavaScript front-end mã nguồn mở và miễn phí để xây dựng giao diện người dùng dựa trên các thành phần UI riêng lẻ

nhớ nhé React là thư viện cũng như thư viện Jquery nhưng nó xịn hơn rất nhiều

Đừng ông nào thấy nó xịn quá xong bảo nó là framework nhé , người ta cười cho đấy :v

cấu trúc dự án

  1. node_modules: lưu trữ thư viện cài đặt
  2. public: lưu image ,css ,js (những cái ta muốn cho ai cũng có thể truy cập)
  3. src : nơi gõ code
  4. file có đuôi .jsx(code react với javascripts) .tsx(code react với typescript)

Reactjs , cơ chế hoạt dộng với Brower

Về cơ bản Brower của chúng ta nó không hiểu code react đâu mà nó chỉ hiểu html css js thôi nên cần phải có trình dịch code (ở đây là vite) và thường thì ta sẽ chạy code ở 2 chế độ

  1. chế độ developemnt (npm run dev)

    dev code (ở đây là javascript)

    compile( dịch code) và lưu trong memory

    Thực ra về bản chất là render ra html css js vào ram nên load cực nhanh sau mỗi lần ta sửa code.

  2. chế độ production

    Code xog rồi , ta muốn build lên chạy thực tế

    compile( dịch code) và lưu cache của Brower

    Chạy với 2 lệ

    • npm run build
    • npm run preview

      • Thực ra về bản chất là render hẳn sang html css js luôn nên khi chạy lên sẽ load cực nhanh nếu chạy thực tế .

  3. ưu và nhược : chat gpt :v

React là Client Side Rendering (CSR)

  • SSR(Server side rendering): Mọi công việc render để tạo ra dữ liệu được xử lí ở phía server(backend)
  • CSR(Client side rendering): Y như thế nhưng xử lí bên Client(fontend)
  • Về bản chất React nó cũng chỉ là HTML CSS JS, fontend rõ ràng :v
